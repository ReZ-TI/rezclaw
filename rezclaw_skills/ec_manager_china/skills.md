# Skill：ec_manager_china（中国电商投放分析 · 淘宝直通车）

**角色 ID**：`ec_manager_china`  
**适用平台**：阿里妈妈统一工作台 [one.alimama.com](https://one.alimama.com) · **关键词推广**（非全站推等其它产品线的细节以后台为准）  
**用途**：指导智能体按统一口径解读投放数据、填分析模板、做问题诊断与优化优先级排序。

> **注意**：当系统配置完成后（Chrome 浏览器 + SSRF 策略），AI 助手可以自动启动浏览器、访问后台、抓取数据并录入智能表格。详见「13. 浏览器自动化场景」。

---

## 何时使用本 Skill

- 用户讨论**淘宝/天猫直通车、关键词推广、计划/单元/关键词**效果分析或周报。
- 需要**解释指标含义**、**判断健康区间**、**排查有展现无点击/有点击无转化/无展现**等。
- 需要按固定结构输出**单元汇总**、**关键词表**或**投放周报**。

---

## 1. 层级结构（必须先建立的心智模型）

```
计划（Campaign）
└── 单元（Ad Group，通常对应一个宝贝/商品）
    └── 关键词
        ├── 手动关键词
        └── 流量智选（系统匹配）
```

分析时说明清楚当前停留在**计划维度 / 单元维度 / 关键词维度**，避免把不同层级指标混在一起谈。

---

## 2. 统计口径（回答「为什么和别报表不一致」时用）

| 项 | 说明 |
|----|------|
| 归因模型 | **末次点击归因** |
| 累计周期 | **15 天**（点击至成交最长归因窗口） |
| 适用 | 新计划需耐心；**重大调整前建议至少观察满 15 天**再下结论 |

后台选时间时，应确认界面展示为「末次点击 + 15 天累计」等与上述一致，再对比数据。

---

## 3. 分析入口与 URL（构造方式）

- **基础地址**：`https://one.alimama.com/index.html`
- **计划列表**：`#!/manage/search`（菜单路径常见为：推广 → 关键词推广）
- **计划/单元详情（含关键词）**：`#!/manage/search-detail?campaignId={计划ID}&adgroupId={单元ID}`

可选查询参数（以当前后台实际支持为准）：`startTime`、`endTime` 等用于固定统计区间。

**计划 ID、单元 ID、商品 ID** 由运营在人类侧维护（表格或内部文档）；智能体**不得编造** ID。若用户未提供，应请用户粘贴后台 URL 或导出字段。

---

## 4. 人类操作步骤（智能体可复述给用户）

1. 进入计划列表：`…/index.html#!/manage/search`
2. 选择时间范围（例如「过去 15 天」），并核对归因与累计周期说明。
3. 点击**计划名称**进入计划维度；在单元中点击**商品名称**进入**关键词维度**。
4. 关键词页 URL 中应可见 `adgroupId=`，便于确认当前单元。

---

## 5. 核心指标（计划/单元层 · 重要性大致排序）

| 指标 | 含义要点 | 健康参考（经验值，非承诺） |
|------|----------|---------------------------|
| 总成交金额 | 投放直接带来的成交额 | 宜 **> 花费**（至少长期目标如此） |
| 投入产出比 ROI | 成交金额 ÷ 花费 | **> 1** 为常见盈亏参考线，越高越好 |
| 点击转化率 | 成交笔数 ÷ 点击量 | 常见关注 **> 1%**（类目差异大） |
| 点击单价 PPC | 花费 ÷ 点击量 | 宜 **低于** 自身出价均值趋势，结合质量分看 |
| 点击率 CTR | 点击量 ÷ 展现量 | 常见关注 **> 2%**（类目/位置差异大） |
| 加购数 / 收藏数 / 总收藏 | 兴趣与潜客信号 | 纵向对比、看趋势 |

说明真实阈值时须强调**类目、客单价、阶段**不同不可硬套；以用户给出的行业或历史数据为准。

---

## 6. 关键词维度常用字段与优化方向

| 维度 | 优化方向（概括） |
|------|------------------|
| 质量分（1–10） | 相关性、创意与落地页体验 |
| 出价 | 结合 ROI、展现量再调 |
| 展现量 | 过低时常见：出价、词量、匹配、时段、预算 |
| 点击量 | 展现 × CTR |
| 花费、PPC | 质量分高通常有利于降低 PPC |
| 成交金额、加购 | 词级盈亏与扩词/删词依据 |

- 统计**完全无展现**的词数。

## 8. 问题诊断清单（按现象）

**有展现、无点击**

- 主图/创意吸引力；标题与词相关性；排名位置（出价与质量分）。

**有点击、无转化**

- 详情页与价格；销量与评价基础；客服与SKU等（本 Skill 仅提示方向）。

**无展现**

- 出价、词体量与匹配；投放时段是否过窄；余额与计划状态。

**其它速查**

| 线索 | 可能处理 |
|------|----------|
| 投放时段极窄 | 在合规与成本允许下逐步扩大有效时段 |
| 余额不足 | 充值或调整预算分配 |
| 冷启动困难 | 了解是否可使用后台「一键起量」等工具（以当前产品为准） |
| 质量分长期偏低（如 <7） | 相关性、创意、落地页与类目一致性 |

---

## 9. 优化优先级（建议话术框架）

- **P0**：时段过窄、预算/余额导致断投、明显阻碍曝光的账户级问题。
- **P1**：清理长期无展现词、对高质高分词适度加价、优化 CTR 明显偏低的创意。
- **P2**：持续测新词、按转化调整出价结构、详情页与承接优化。

具体执行顺序由人类根据库存、利润与活动节奏决定。

---

## 10. 注意事项（避免误导用户）

- **数据延迟**：点击偏实时，成交归因最长 15 天；新计划忌过早大幅否定。
- **误区**：勿单看展现或单看点击；勿只调价不改创意与承接；勿在极窄时段下结论。
- **报表口径**：计划管理页与「报表/数据」类模块可能不一致时，**以官方报表或人类指定口径为准**。

## 12. 智能体行为约束

- **不虚构** 花费、成交、ROI、质量分等；用户未给数据时输出**模板 + 待填项**，并说明需从后台复制或导出。
- **不泄露** 用户粘贴的账户 ID、内部商品 ID；不在无关会话中复述。
- 平台功能名、菜单位置随阿里妈妈改版可能变化；遇阻时建议用户以**当前后台界面**为准并更新本 Skill 的人类维护版。

## 13. 浏览器自动化场景（OpenClaw 定时任务）

当系统完成以下配置后，AI 助手可实现全自动数据抓取与录入：

### 13.1 前置条件

| 配置项 | 说明 |
|--------|------|
| Chrome 浏览器 | 启动参数：`--remote-debugging-port=28800` |
| OpenClaw SSRF 策略 | 允许 `one.alimama.com`、`sycm.taobao.com` |
| 登录态 | 阿里妈妈后台保持登录 |

### 13.2 自动化流程

```
定时任务触发（每天 10:00 AM）
    ↓
启动 Chrome（如未运行）
    ↓
访问阿里妈妈后台 → 抓取昨日数据
    ↓
录入企业微信智能表格
    ↓
记录执行日志
```

### 13.3 智能表格录入

使用 `wecom_mcp` 工具：
- `smartsheet_get_sheet` - 获取子表信息
- `smartsheet_get_fields` - 获取字段结构
- `smartsheet_add_records` - 添加数据记录

**表格 URL 示例**：
`https://doc.weixin.qq.com/smartsheet/s3_ASEAnQYSAF4CNrUvCw1tCQPSNs0xy`

### 13.4 配置示例

```json
// openclaw.json
{
  "browser": {
    "enabled": true,
    "ssrfPolicy": {
      "dangerouslyAllowPrivateNetwork": true,
      "allowedHostnames": [
        "one.alimama.com",
        "sycm.taobao.com"
      ]
    }
  }
}
```
