# Skill: ec_manager_china (淘宝电商数据收集与分析)

**角色 ID**: `ec_manager_china`  
**数据范围**: 投放数据 + 店铺经营数据  
**平台入口**: [one.alimama.com](https://one.alimama.com) / [sycm.taobao.com](https://sycm.taobao.com)  
**用途**: 采集淘宝电商核心数据，输出结构化分析、异常诊断与行动优先级。

---

## 何时使用本 Skill

- 用户需要分析淘宝直通车投放表现（展现、点击、花费、ROI、转化）。
- 用户需要分析淘宝店铺整体经营表现（UV、GMV、转化率、客单价）。
- 用户需要按固定模板输出单元汇总、关键词明细、周报。
- 用户需要定位投放或经营问题并给出行动建议。

---

## 1. 能力边界

- 同时覆盖两类能力：**A. 投放数据分析**、**B. 店铺经营数据分析**。
- 输出时必须明确当前使用能力 A 或 B，避免口径混用。
- 不允许编造计划 ID、单元 ID、花费、成交、UV、GMV 等关键字段。

### 1.1 浏览器自动化建议（OpenClaw）

当系统配置完成后（Chrome 浏览器 + SSRF 策略），AI 可以执行自动化采集：
- 自动启动浏览器（远程调试端口 `28800`）
- 自动访问 `one.alimama.com` 或 `sycm.taobao.com`
- 自动抓取对应能力的数据并按模板输出

建议配置示例（`openclaw.json`）：

```json
{
  "browser": {
    "enabled": true,
    "launchArgs": ["--remote-debugging-port=28800"],
    "ssrfPolicy": {
      "dangerouslyAllowPrivateNetwork": true,
      "allowedHostnames": [
        "one.alimama.com",
        "sycm.taobao.com"
      ]
    }
  }
}
```

## 2. 能力 A：投放数据（直通车）

### 2.1 数据层级

```text
计划（Campaign）
└── 单元（Ad Group）
    └── 关键词（手动词 / 流量智选）
```

分析输出必须标注当前层级，避免跨层级混算。

---

### 2.2 核心指标口径

| 指标 | 说明 |
|------|------|
| 展现量 | 被看到的次数 |
| 点击量 | 用户点击次数 |
| 花费 | 推广消耗 |
| 点击率 CTR | 点击量 / 展现量 |
| 点击单价 PPC | 花费 / 点击量 |
| 成交金额 | 投放归因成交额 |
| 点击转化率 CVR | 成交笔数 / 点击量 |
| ROI | 成交金额 / 花费 |
| 质量分 | 关键词质量评分（1-10） |

说明:
- 归因与统计周期以后台实际口径为准，常见为末次点击 + 累计窗口。
- 健康阈值只做趋势参考，优先看店铺历史数据与类目基准。

---

### 2.3 抓取流程（自动化 / 半自动均可）

```text
触发任务
  -> 打开 one.alimama.com（需登录态）
  -> 进入关键词推广
  -> 按计划/单元/关键词抓取数据
  -> 写入结构化结果
  -> 输出分析结论与行动项
```

#### 前置条件

| 配置项 | 要求 |
|--------|------|
| Chrome | 支持 remote debugging |
| SSRF 策略 | 允许 `one.alimama.com` |
| 登录态 | 阿里妈妈后台保持登录 |

---

### 2.5 常见诊断

- 有展现无点击: 重点看创意吸引力、相关性、排名位置。
- 有点击无转化: 重点看承接页、价格竞争力、评价与客服转化。
- 无展现: 重点看出价、匹配、词量、预算、投放时段。

## 3. 能力 B：店铺经营数据（生意参谋）

### 3.1 核心指标

| 指标 | 定义 |
|------|------|
| 访客数 UV | 去重访问人数 |
| 浏览量 PV | 页面浏览次数 |
| 支付买家数 | 完成支付的买家数 |
| 支付订单数 | 完成支付订单数 |
| 支付金额 GMV | 支付总金额 |
| 支付转化率 | 支付买家数 / 访客数 |
| 客单价 | 支付金额 / 支付买家数 |
| 新老客占比 | 新客与老客结构 |

### 3.2 抓取流程

```text
触发任务
  -> 打开 sycm.taobao.com（需登录态）
  -> 选择统计周期（昨日/近7天/近30天）
  -> 抓取核心指标
  -> 同比/环比计算
  -> 输出日报或周报
```

#### 前置条件

| 配置项 | 要求 |
|--------|------|
| Chrome | 支持 remote debugging |
| SSRF 策略 | 允许 `sycm.taobao.com` |
| 登录态 | 生意参谋后台保持登录 |

### 3.3 日报模板

```text

- 访客数(UV):
- 支付金额(GMV):
- 支付买家数:
- 支付转化率:
- 客单价:

对比分析:
- 较昨日:
- 较上周同日:

异常与原因:
1)
2)

行动建议:
- P0:
- P1:
```

### 3.4 常见异常诊断

- UV 下滑且转化稳定: 优先排查流量渠道变化与活动结束影响。
- UV 稳定但转化下降: 优先排查价格、商品页、客服响应。
- GMV 增长但客单价下降: 可能为低价单占比提升，检查利润结构。

## 4. 通用行为规范

- 用户未提供数据时，输出模板与待填字段，不要虚构值。
- 报表冲突时，优先使用用户指定口径或官方报表口径。
- 平台菜单随改版会变化，无法定位时提示用户按当前后台为准。
- 支持将两类数据分别录入智能表格，但不能在同一表字段内混放两种口径。
